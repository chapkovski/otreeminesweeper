Thank you for your participation in this experiment. Before proceeding to the game portion of the study, you will need to answer a few questions to indicate you understand how the game works, so please read the following instructions carefully.


Game Overview

You have the opportunity to earn money in this experiment based on your decisions and the procedures described in these instructions. For this experiment, you will be participating in two tasks that center around a modified version of the computer game Minesweeper. Details of the game will be explained below, and you will play several practice rounds to become familiar with the game.


Each Minesweeper game starts out with a grid of unmarked squares (see figure 1).




After left-clicking one of these squares, some of the squares will disappear, some will remain blank, and some will have numbers on them (see figure 2).



It is your role to use the numbers to figure out which blank squares have bombs (referred to as mines in the original version of the game). A number on a square refers to the number of bombs that are currently touching that square. For example, if there is a 1 on a square and only one square adjacent, you know there is a bomb on the adjacent square (see figure 3)



However, if there is a square with the number 2, you know there must be 2 adjacent bombs as seen in figure 4.




The mouse is the only tool that you'll need to play Minesweeper. The left mouse button is used to reveal squares that don't contain bombs, while the right mouse button is used to mark squares that contain bombs. Incorrectly clicking a bomb with a left click will result in a financial penalty which will be explained later. Each time you mark a potential bomb with a right-click, a red exclamation point will appear on the square so that you can keep track of all the bombs on the grid. The grid will begin to open as you reveal all the squares without bombs (see figure 5).
